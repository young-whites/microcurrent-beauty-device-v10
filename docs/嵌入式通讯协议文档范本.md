## 三、帧格式定义

### 3.1 主协议帧格式（自定义）

```
[帧头] [len] [addrH] [addrL] [type] [state] [func] [para1] ... [paraN] [crcH] [crcL]
```

| 字段 | 字节数 | 值/范围 | 说明 |
|------|--------|---------|------|
| 帧头 | 2 | `[HEAD1] [HEAD2]` | 固定标识，按项目定义 |
| 长度 (len) | 1 | 0x05~0xFF | addrH ~ paraN 的总字节数，len = 5 + 参数数量 |
| 设备地址高字节 (addrH) | 1 | `DEVICE_ID_H` | 设备固定地址高字节 |
| 设备地址低字节 (addrL) | 1 | `DEVICE_ID_L` | 设备固定地址低字节 |
| 类型 (type) | 1 | `FRAME_TYPE_ACT`=动作, `FRAME_TYPE_GET`=查询 | 命令分类 |
| 状态 (state) | 1 | `FRAME_STATE_ACK`=应答, `FRAME_STATE_ASK`=请求 | 通信方向标识 |
| 功能码 (func) | 1 | `0x01`~`0xFF` | 具体功能编号 |
| 参数 (para) | 0~N | 可变 | 功能相关参数，按需定义 |
| 校验 (crcH/crcL) | 2 | CRC16-XXXX | 校验范围：len ~ paraN 共 (len+1) 字节，**高字节在前** |

> ⚠️ **len 字段计算要点**：len 包含地址 2 字节 + 类型 1 字节 + 状态 1 字节 + 功能码 1 字节 + 参数 N 字节 = 5 + N。
> - 无参数指令：len = 0x05
> - 单参数指令：len = 0x06
> - 双参数指令：len = 0x07

### 3.2 设备固件宏定义

```c
/* 主板设备码以及版本号 */
#define DEVICE_ID_H     0xXX
#define DEVICE_ID_L     0xXX
#define DEVICE_VERSION  0x01
```

### 3.3 帧参数宏定义

```c
#define FRAME_HEAD1     0xXX    // Frame header 1
#define FRAME_HEAD2     0xXX    // Frame header 2
#define FRAME_TYPE_ACT  0xXX    // Frame type: action command
#define FRAME_TYPE_GET  0xXX    // Frame type: parameter get
#define FRAME_STATE_ASK 0xXX    // Frame state: host request
#define FRAME_STATE_ACK 0xXX    // Frame state: device response
```

### 3.4 CRC 校验规则

| 项目 | 值 |
|------|-----|
| 算法 | CRC16-MODBUS / CRC16-CCITT / CRC16-X25（按需选择） |
| 多项式 | [0x8005 / 0x1021 / ...] |
| 初始值 | [0xFFFF / 0x0000 / ...] |
| 输入反转 | [True / False] |
| 输出反转 | [True / False] |
| 异或值 | [0x0000 / ...] |
| 计算范围 | len ~ paraN（不含帧头 55 AA 和 CRC 本身），共 (len+1) 字节 |
| 字节序 | **高字节在前 / 低字节在前**（必须明确约定） |
| 示例 | 计算结果 `0x1A75` → 帧中排列 `1A 75`（Big-Endian 示例） |

> ⚠️ 不同协议的 CRC 字节序可能不同（如标准 Modbus 为小端，自定义协议可能为大端），务必在文档中明确标注。

### 3.5 len 字段计算规则

```
len = addrH(1) + addrL(1) + type(1) + state(1) + func(1) + 参数数量
len = 5 + 参数数量

示例：
- 无参数指令（读版本号）：len = 5 + 0 = 0x05
- 单参数指令（能量调节）：len = 5 + 1 = 0x06
- 双参数指令（版本号应答）：len = 5 + 2 = 0x07
```

## 六、应答机制

### 6.1 成功应答

| 场景 | state 字段 | 帧结构变化 |
|------|-----------|-----------|
| 请求（上位机→设备） | `FRAME_STATE_ASK` | 保持原始功能码和参数 |
| 应答（设备→上位机） | `FRAME_STATE_ACK` | state 改为 ACK 值，其余字段回传 |

### 6.2 错误应答（如需定义）

| 错误码 | 含义 | 说明 |
|--------|------|------|
| `0x00` | 成功 | — |
| `0x01` | 参数错误 | 参数越界或格式不合法 |
| `0x02` | 不支持的功能码 | func 未实现 |
| `0x03` | 设备忙 | 当前状态不允许执行 |
| `0xFF` | 未知错误 | — |

> 错误应答帧格式（扩展定义，待确认）：

```

3. [操作2]:  [HEAD1] [HEAD2] 06 [addr] [type] [ASK] 01 [para] [crc]  ← 设置参数
4. [操作3]:  [HEAD1] [HEAD2] 06 [addr] [type] [ASK] 02 [para] [crc]  ← 启动功能
5. [操作4]:  [HEAD1] [HEAD2] 05 [addr] [type] [ASK] 03 [crc]   ← 读取结果
```
