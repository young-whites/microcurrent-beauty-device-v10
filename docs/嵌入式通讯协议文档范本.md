## 三、帧格式定义

### 3.1 主协议帧格式（自定义）

```
[帧头] [长度] [地址] [类型] [状态] [功能码] [参数...] [校验]
```

| 字段 | 字节数 | 值/范围 | 说明 |
|------|--------|---------|------|
| 帧头 | 2 | `0xAA 0xBB` | 固定标识，按项目定义 |
| 长度 (len) | 1 | 0x00~0xFF | 数据域字节数（帧头之后、校验之前） |
| 地址 (addr) | 2 | `[addrH] [addrL]` | 设备地址，大端序 |
| 类型 (type) | 1 | `0x44`=动作, `0x45`=查询 | 命令分类 |
| 状态 (state) | 1 | `0x01`=应答, `0x02`=请求 | 通信方向标识 |
| 功能码 (func) | 1 | `0x01`~`0xFF` | 具体功能编号 |
| 参数 (para) | 0~N | 可变 | 功能相关参数，按需定义 |
| 校验 (CRC) | 2 | CRC16-XXXX | 校验范围：长度字段 ~ 最后参数 |

### 3.2 CRC 校验规则

| 项目 | 值 |
|------|-----|
| 算法 | CRC16-MODBUS / CRC16-CCITT / CRC16-X25（按需选择） |
| 计算范围 | len ~ paraN（不含帧头和CRC本身） |
| 字节序 | **高字节在前 / 低字节在前**（必须明确约定） |
| 示例 | 计算结果 `0x1A75` → 帧中排列 `1A 75`（Big-Endian 示例） |

> ⚠️ 不同协议的 CRC 字节序可能不同（如标准 Modbus 为小端，自定义协议可能为大端），务必在文档中明确标注。

## 六、应答机制

### 6.1 成功应答

| 场景 | state 字段 | 帧结构变化 |
|------|-----------|-----------|
| 请求（上位机→设备） | `0x02` | 保持原始功能码和参数 |
| 应答（设备→上位机） | `0x01` | state 改为 0x01，其余字段回传 |

### 6.2 错误应答（如需定义）

| 错误码 | 含义 | 说明 |
|--------|------|------|
| `0x00` | 成功 | — |
| `0x01` | 参数错误 | 参数越界或格式不合法 |
| `0x02` | 不支持的功能码 | func 未实现 |
| `0x03` | 设备忙 | 当前状态不允许执行 |
| `0xFF` | 未知错误 | — |
