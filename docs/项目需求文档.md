### 3.4 NNC6521 双芯片引脚映射

本项目使用两颗 NNC6521 微电流刺激芯片，通过软件 SPI（GPIO Bit-Bang）驱动。

| 功能 | Chip 1（NNC6521_1） | Chip 2（NNC6521_2） |
|------|---------------------|---------------------|
| MOSI | PC9 | PB13 |
| CSN | PC8 | PB12 |
| SCLK | PC7 | PB11 |
| MISO | PA8 | PB14 |
| CHIP_EN | PA12 | PC6 |
| INTB（中断） | PA11 | PB15 |

**SPI 时序**: CPOL=0, CPHA=0, MSB First  
**写协议**: 4 字节 `[addr, cmd(0x80/0xC0), 0x11, data]`  
**读协议**: 3 字节 `[addr, cmd(0x00/0x40), dummy]` → 返回数据  
**cmd 含义**: `0x80` = 普通寄存器写, `0xC0` = 波形寄存器写, `0x00` = 普通寄存器读, `0x40` = 波形寄存器读

## 五、通信协议

### 5.1 帧格式

```
+--------+--------+------+--------+--------+------+-------+------+-------+-------+
| HEAD1  | HEAD2  | LEN  | ADDR_H | ADDR_L | TYPE | STATE | FUNC | PARAMS| CRC16 |
| 0x55   | 0xAA   | var  | 0x60   | 0x66   | var  | var   | var  | var   | 2B    |
+--------+--------+------+--------+--------+------+-------+------+-------+-------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| HEAD1 | 1B | 固定 0x55 |
| HEAD2 | 1B | 固定 0xAA |
| LEN | 1B | 从 LEN 字段本身到 PARAMS 末尾的字节数（不含 HEAD1/HEAD2，不含 CRC） |
| ADDR_H | 1B | 设备地址高字节（0x60） |
| ADDR_L | 1B | 设备地址低字节（0x66） |
| TYPE | 1B | 帧类型：0x44=动作命令，0x45=参数查询 |
| STATE | 1B | 帧状态：0x02=主机请求(ASK)，0x01=设备应答(ACK) |
| FUNC | 1B | 功能码（0x01~0x09） |
| PARAMS | NB | 功能参数（长度由 LEN 决定） |
| CRC16 | 2B | CRC16-MODBUS，计算范围为 LEN ~ PARAMS 末尾 |

### 5.2 CRC16-MODBUS 算法

- 初始值: `0xFFFF`
- 多项式: `0xA001`（反转）
- 计算范围: LEN 字段到 PARAMS 最后一个字节

### 5.3 解码状态机

```
IDLE → HEAD2 → LENGTH → ADDR_H → ADDR_L → PAYLOAD → CRC_H → CRC_L
  ↑                                                             │
  └─────────────────── 校验失败 / 正常完成 ──────────────────────┘
```

### 5.4 串口缓冲区配置

| 参数 | 值 |
|------|-----|
| 接收环形缓冲区 | 512 字节 |
| 发送缓冲区 | 64 字节 |
| 最大帧长度 | 64 字节 |
| 命令解码缓冲区 | 32 字节 |
| UART 接收缓冲区 | 1024 字节（rt_device buf） |

## 六、功能需求（分模块）

### 6.1 手柄切换模块（功能码 0x01）

**功能描述**: 切换当前活动手柄。

| 手柄 ID | 手柄名称 | 功能 |
|---------|---------|------|
| 0x0A | 手柄 A | 微电流 + 加热 |
| 0x0B | 手柄 B | 微电流 + 加热 |
| 0x0C | 手柄 C | 微电流 + 泵 |

**请求参数**:
| 字段 | 长度 | 说明 |
|------|------|------|
| handle_id | 1B | 目标手柄 ID（0x0A/0x0B/0x0C） |

**应答参数**:
| 字段 | 长度 | 说明 |
|------|------|------|
| handle_id | 1B | 已切换到的手柄 ID |

**行为**:
1. 切换前自动停止当前波形输出
2. 更新全局 `current_handle` 字段
3. 校验 handle_id 合法性，非法则返回错误码 0x01

### 6.2 电流控制模块（功能码 0x02）

**功能描述**: 设置指定手柄的微电流输出百分比。

**请求参数**:
| 字段 | 长度 | 说明 |
|------|------|------|
| percent | 1B | 电流百分比（0~100），映射到当前波形的电流范围 |
| handle_id | 1B | 目标手柄 ID |

**应答参数**:
| 字段 | 长度 | 说明 |
|------|------|------|
| percent | 1B | 设置的百分比 |
| handle_id | 1B | 目标手柄 ID |

**行为**:
1. 保存到目标手柄的 `current_percent` 字段
2. 若目标手柄为当前活动手柄，立即更新 NNC6521 输出
3. 百分比映射公式: `actual_current = min + (max - min) × percent / 100`

### 6.3 温度控制模块（功能码 0x03）

**功能描述**: 设置手柄加热目标温度。

**请求参数**:

**行为**:
1. 温度范围限制: 0~41℃
2. 仅手柄 A 和 B 支持加热（手柄 C 不支持）


| 1 | 30 | 80 |
| 2 | 30 | 80 |
| 3 | 20 | 60 |
| 4 | 30 | 80 |
| 5 | 30 | 80 |
| 6 | 20 | 60 |
| 7 | 15 | 50 |
| 8 | 15 | 40 |
| 9 | 10 | 30 |

### 7.3 波形生成方式

NNC6521 支持三种波形生成方式：

1. **预加载波形 (Preloaded)**: 芯片内置的正弦/脉冲/三角波，通过寄存器选择
2. **自定义波形 (Customized SPI)**: 通过 SPI 写入归一化浮点数组，由芯片 DAC 输出
3. **幅度调制 (Amplitude Modulation)**: 包络线 + 载波的调制输出

## 十、NNC6521 驱动层

### 10.1 芯片初始化流程

```
1. nnc6521_gpio_init()         → 初始化所有 GPIO 引脚
2. nnc6521_init(chip_id)       → 上电序列:
   a. CHIP_EN = LOW（复位）
   b. 延时 ~10ms
   c. CHIP_EN = HIGH（使能）
   d. 延时 ~10ms
   e. 写 PMU 寄存器: 复位波形发生器
   f. 配置时钟: PCLK 分频 = 1（2MHz）
   g. 全局驱动使能
```

### 10.2 波形发生器配置流程

```
1. 禁用目标通道 AWG
2. 使能模拟通道 (ANA_ENABLE_REG)
3. 写入配置寄存器组:
   - 采样点数 (POINT_CONFIG)
   - 配置寄存器 (CONFIG_REG0): rest/negative/silent/sourceB/multi_electrode/continue
   - 静默时间 (SILENT_CLK): 24-bit
   - 正半波间隔 (HALF_WAVE_CLK_PNT): 16-bit
   - 负半波间隔 (NEG_HALF_WAVE_CLK_PNT): 16-bit
   - 驱动电流寄存器 2 (DRIVE_REG_CTRL2)
   - 死区时间 (REST_CLK): 16-bit
   - 中断配置 (INT_NUM_WAVE, INT_REG01~03)
4. 若为 SPI 波形:
   a. 通过 OTP 校准数据计算电流
   b. 生成缩放波形数组 (16-bit → 8-bit)
   c. 逐点写入波形数据到 Waveform RAM
5. 配置交替模式寄存器
6. 使能 AWG
```

### 10.3 电流校准系统

- 校准数据存储在 NNC6521 OTP 内存中
- 25 个校准点（范围 3~25）
- CH1 校准基地址: `0x00`
- CH2 校准基地址: `0x32`
- 校准流程: 读取 OTP Dref → 计算 Ival = (Ireq × 1000) / Dref → 限制最大 4096

### 10.4 安全检测

#### 10.4.1 导联脱落检测 (LOD - Lead-Off Detection)

- 检测模式: 高电平检测
- 检测通道: CH1 + CH2 同时（LO_ALL）
- 寄存器组: `0x30~0x3B`（LEAD_OFF_xxx）
- 中断使能位: `LEAD_OFF_INT.lead_off_int_en`
- 配置参数: 阈值（16-bit）、延迟、目标值

#### 10.4.2 短路检测 (SCD - Short-Circuit Detection)

- 双通道独立检测
- 通过 DAC 阈值比较器实现
- 寄存器组: `ANA_INT_SIMx_xxx`, `ANA_INT_CHx_INT_NUMBER`
- 中断清零: 写 `ANA_INTR_SIM_CL = 0x03`

| 指标 | 要求 |
|------|------|
