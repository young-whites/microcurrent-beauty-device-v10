# DJM-V10 微电流美容仪 - 通信协议文档

> **版本**: V4.0  
> **日期**: 2026-06-11  
> **适用平台**: STM32F103 / RT-Thread / HAL 库

---

> [!important] CRC字节序说明
> CRC16-MODBUS 计算结果为双字节，**在帧中以高字节在前、低字节在后的顺序存放**（Big-Endian）。
> 例如计算结果为 `0x0F48`，则帧中排列为 `0F 48`（先 crcH=`0x0F`，后 crcL=`0x48`）。
> ⚠️ 本文档所有完整帧示例均经 CRC 校验确认，可直接用于开发测试。

## 三、帧格式定义

### 3.1 主协议帧格式

```
55 AA [len] [addrH] [addrL] [type] [state] [func] [para1] ... [paraN] [crcH] [crcL]
```

| 字段 | 字节数 | 值/范围 | 说明 |
|------|--------|---------|------|
| 帧头 | 2 | `FRAME_HEAD1`(0x55) `FRAME_HEAD2`(0xAA) | 固定标识 |
| 长度 (len) | 1 | 0x05~0xFF | addrH ~ paraN 的总字节数，len = 4 + 参数数量 + 2（地址） |
| 设备地址高字节 (addrH) | 1 | `DEVICE_ID_H`(0x60) | 设备固定地址高字节 |
| 设备地址低字节 (addrL) | 1 | `DEVICE_ID_L`(0x66) | 设备固定地址低字节 |
| 类型 (type) | 1 | `FRAME_TYPE_ACT`(0x44)=动作命令, `FRAME_TYPE_GET`(0x45)=参数获取 | 命令分类 |
| 状态 (state) | 1 | `FRAME_STATE_ACK`(0x01)=设备应答, `FRAME_STATE_ASK`(0x02)=上位机请求 | 通信方向标识 |
| 功能码 (func) | 1 | `0x01`~`0x08` | 具体功能编号 |
| 参数 (para) | 0~N | 可变 | 功能相关参数，按需定义 |
| 校验 (crcH/crcL) | 2 | CRC16-MODBUS | 校验范围：len ~ paraN 共 (len+1) 字节，**高字节在前** |

### 3.2 设备固件宏定义

```c
/* 主板设备码以及版本号 */
#define DEVICE_ID_H     0x60
#define DEVICE_ID_L     0x66
#define DEVICE_VERSION  0x01
```

### 3.3 帧参数宏定义

```c
#define FRAME_HEAD1     0x55    // Frame header 1
#define FRAME_HEAD2     0xAA    // Frame header 2
#define FRAME_TYPE_ACT  0x44    // Frame type: action command
#define FRAME_TYPE_GET  0x45    // Frame type: parameter get
#define FRAME_STATE_ASK 0x02    // Frame state: host request
#define FRAME_STATE_ACK 0x01    // Frame state: device response
```

### 3.4 CRC 校验规则

| 项目 | 值 |
|------|-----|
| 算法 | CRC16-MODBUS |
| 多项式 | 0x8005 |
| 初始值 | 0xFFFF |
| 输入反转 | True（RefIn） |
| 输出反转 | True（RefOut） |
| 异或值 | 0x0000 |
| 计算范围 | len ~ paraN（不含帧头 55 AA 和 CRC 本身），共 (len+1) 字节 |
| 字节序 | **高字节在前，低字节在后**（Big-Endian） |
| 示例 | 计算结果 `0x0F48` → 帧中排列 `0F 48` |
| 固件版本 | `DEVICE_VERSION = 0x01`（V1.0） |

### 3.5 len 字段计算规则

```
len = addrH(1) + addrL(1) + type(1) + state(1) + func(1) + 参数数量
len = 5 + 参数数量

示例：
- 无参数指令（读版本号）：len = 5 + 0 = 0x05
- 单参数指令（能量调节）：len = 5 + 1 = 0x06
- 双参数指令（版本号应答）：len = 5 + 2 = 0x07
```

## 四、手柄系统说明

### 4.1 手柄功能矩阵

| 手柄 | 编号码 | 微电流输出 | 加热功能 | 气泵控制 | 说明 |
|------|--------|-----------|---------|---------|------|
| A | `0x0A` | ✅ | ✅ | ❌ | 默认手柄，加热+微电流 |
| B | `0x0B` | ✅ | ✅ | ❌ | 功能同 A |
| C | `0x0C` | ✅ | ❌ | ✅ | 微电流+气泵 |

### 4.2 手柄切换行为

- **默认状态**：上电后处于手柄 A 界面
- **切换时保存**：切换到其他手柄时，当前手柄的参数值（能量、温度、泵速）自动保存
- **切回恢复**：切回某手柄时，恢复该手柄之前保存的参数值
- **独立调节**：每个手柄页面只可调节自己的参数，不可跨手柄调节
- **切换时复位**：切换手柄后设备自动停止当前输出状态

### 4.3 参数范围总览

| 参数 | 范围 | 单位 | 适用手柄 | 说明 |
|------|------|------|----------|------|
| 电流能量 | 0~100 | % | A, B, C | 0=关闭，1~100=输出百分比 |
| 温度 | 0~41 | ℃ | A, B | 0=关闭加热，最大 41℃ |
| 气泵转速 | 0~100 | % | C | 0=关闭，1~100=转速百分比 |

## 五、功能码定义

### 5.1 动作命令（type=FRAME_TYPE_ACT=0x44）

| 功能码 | 名称 | 参数定义 | 适用手柄 | 说明 |
|--------|------|----------|----------|------|
| `0x01` | 手柄切换 | para[0]=手柄编号(0x0A/0x0B/0x0C) | 全局 | 切换当前工作手柄 |
| `0x02` | 电流能量调节 | para[0]=能量等级(0~100) | A, B, C | 调节微电流输出强度 |
| `0x03` | 温度调节 | para[0]=目标温度(0~41℃) | A, B | 调节加热温度 |
| `0x04` | 气泵转速调节 | para[0]=转速等级(0~100) | C | 调节气泵转速 |
| `0x05` | 开始/暂停 | para[0]=动作(0=暂停, 1=开始) | 全局 | 控制治疗流程启停 |
| `0x06` | 远程升级 | para[0]=升级类型(0=开始, 1=确认, 2=取消) | 全局 | OTA 固件升级控制 |
| `0x07` | 老化模式 | para[0]=动作(0=退出, 1=进入) | 全局 | 工厂老化测试模式 |
| `0x08` | 读取版本号 | 无参数 | 全局 | 查询固件版本 |

## 六、指令详细定义（含已验证 CRC）

> 以下所有完整帧的 CRC 均已通过 CRC16-MODBUS 算法验证，可直接用于开发测试。
> 设备地址：`0x60 0x66`（`DEVICE_ID_H`=0x60, `DEVICE_ID_L`=0x66）

### 6.1 手柄切换（func=0x01）

**方向**: APP → 设备

**参数定义**:

| 参数 | 偏移 | 长度 | 类型 | 范围 | 说明 |
|------|------|------|------|------|------|
| para[0] | 0 | 1 byte | uint8 | 0x0A/0x0B/0x0C | 手柄编号（A=0x0A, B=0x0B, C=0x0C） |

**请求帧（APP→设备, state=FRAME_STATE_ASK, len=0x06）**:

| 操作 | 完整帧（含 CRC） |
|------|-------------------|
| 切换至手柄 A | `55 AA 06 60 66 44 02 01 0A 0F 48` |
| 切换至手柄 B | `55 AA 06 60 66 44 02 01 0B CF 89` |
| 切换至手柄 C | `55 AA 06 60 66 44 02 01 0C 0D C8` |

**应答帧（设备→APP, state=FRAME_STATE_ACK, len=0x06）**:

| 操作 | 完整帧（含 CRC） |
|------|-------------------|
| 手柄 A 切换成功 | `55 AA 06 60 66 44 01 01 0A 0F B8` |
| 手柄 B 切换成功 | `55 AA 06 60 66 44 01 01 0B CF 79` |
| 手柄 C 切换成功 | `55 AA 06 60 66 44 01 01 0C 0D 38` |

**说明**:
- 切换手柄后设备自动停止当前输出状态
- 各手柄参数独立保存，切回时恢复之前的参数值
- 若手柄编号无效（非 0x0A/0x0B/0x0C），设备返回参数错误应答

|------|-------------------|

## 七、应答机制

### 7.1 成功应答

| 场景 | state 字段 | 帧结构变化 |
|------|-----------|-----------|
| 请求（APP→设备） | `FRAME_STATE_ASK`(0x02) | APP 发送，保持原始功能码和参数 |
| 应答（设备→APP） | `FRAME_STATE_ACK`(0x01) | 设备回复，state 改为 0x01，其余字段回传 |

### 7.2 错误应答

| 错误码 | 含义 | 说明 |
|--------|------|------|
| `0x00` | 成功 | 操作正常完成 |
| `0x01` | 参数错误 | 参数越界或格式不合法 |
| `0x02` | 不支持的功能码 | func 未实现 |
| `0x03` | 设备忙 | 当前状态不允许执行（如治疗中不允许升级） |
| `0xFF` | 未知错误 | 系统异常 |

> 错误应答帧格式（扩展定义，待确认）：
> `55 AA 07 60 66 44 01 [func] [errorCode] [crcH] [crcL]`

## 八、通信时序

### 8.1 典型治疗流程

```
1. 上电初始化（默认手柄 A）
2. 读取版本号:       55 AA 05 60 66 44 02 08 7D DE        ← 确认设备在线
3. 设置能量:         55 AA 06 60 66 44 02 02 32 2D 49      ← 能量设为50
4. 设置温度:         55 AA 06 60 66 44 02 03 25 B3 08      ← 温度设为37℃
5. 开始治疗:         55 AA 06 60 66 44 02 05 01 08 0B      ← 启动治疗
6. 实时调节能量:     55 AA 06 60 66 44 02 02 50 [crc]      ← 能量调为80
7. 暂停治疗:         55 AA 06 60 66 44 02 05 00 C8 CA      ← 暂停

--- 切换到手柄 C ---
8. 切换手柄 C:       55 AA 06 60 66 44 02 01 0C 0D C8      ← 切换手柄
9. 设置气泵:         55 AA 06 60 66 44 02 04 32 8D 4A      ← 气泵50%
10. 设置能量:        55 AA 06 60 66 44 02 02 32 [crc]      ← 能量50%
11. 开始治疗:        55 AA 06 60 66 44 02 05 01 [crc]      ← 启动

--- 切回手柄 A ---
12. 切换手柄 A:      55 AA 06 60 66 44 02 01 0A 0F 48      ← 切回A，恢复之前的参数
```

### 8.2 时序约束

| 约束项 | 值 | 说明 |
|--------|-----|------|
| 请求-应答超时 | 500 ms | 超过此时间未应答视为通信失败 |
| 最小发送间隔 | 50 ms | 连续两帧之间的最小间隔 |
| 重发次数 | 3 次 | 超时后最大重发次数 |
| 设备初始化时间 | 1000 ms | 上电后需等待此时间才可通信 |

### 8.3 注意事项

- 所有动作命令（type=FRAME_TYPE_ACT）均遵循「请求→应答」模式，APP 发送请求后必须等待设备应答
- 读取版本号（func=0x08）是唯一无参数指令，len 固定为 0x05
- 远程升级（func=0x06）为预留功能，固件数据传输细节待后续定义
- UART2 仅供调试日志输出，不参与协议通信
- 手柄 A/B 支持加热，手柄 C 不支持加热（发温度指令无效）
- 手柄 C 支持气泵，手柄 A/B 不支持气泵（发气泵指令无效）
